# Context Directory

All inter-agent shared state lives here. Files are grouped into subdirectories by producer.

## Directory Structure

```
context/
├── research_context.md          # project specification (human input)
├── evaluation_criteria.md       # 8-criterion scoring rubric (human input)
├── evaluator_feedback.md        # current evaluator feedback (overwritten each iter)
├── model_equations.md           # all derivations and propositions
├── model_verifier_report.md     # Model Verifier assessment
├── improvement_task_queue.md    # QA improvement tasks
├── loop_state.md                # iteration counter, score, exit condition
├── phase_state.md               # current pipeline position (shell script)
├── human_feedback_phase1.md     # optional human corrections to threat map
│
├── literature/                  # Literature Guardian outputs
│   ├── threat_map_v1.md         # initial Phase 1 threat map (permanent)
│   ├── threat_map.md            # accumulated threat map with changelog
│   ├── threat_map_final.md      # clean, definitive threat map (Phase 3)
│   ├── constraints.md           # what the literature has/hasn't addressed
│   ├── notes.md                 # structured paper summaries
│   ├── review.md                # prose review for LaTeX conversion
│   ├── review_post_review.md    # post-QA literature additions
│   └── search_log.md            # cumulative search query log
│
├── planning/                    # Research Director outputs
│   ├── research_plan.md         # evolves with changelog per iteration
│   ├── research_plan_final.md   # consolidated research program (Phase 3)
│   ├── paper_structure.md       # section map for Paper Writer
│   ├── task_queue.md            # task list for downstream agents
│   └── novelty_claims.md        # contribution claims for verification
│
├── archives/                    # iteration snapshots (auto-generated)
│   └── evaluator_feedback_i{N}.md
│
└── self_reviews/                # Phase 4 QA review reports
    ├── review_theory.md
    ├── review_presentation.md
    └── review_framing.md
```

## Human Inputs (read-only — never modified by agents)
- `research_context.md` — project specification, scope constraints
- `evaluation_criteria.md` — 8-criterion scoring rubric

## Loop State (modified each Phase 2 iteration)
- `planning/research_plan.md` — evolves with changelog per iteration
- `literature/threat_map.md` — accumulated threat map with changelog
- `loop_state.md` — iteration counter, score, exit condition
- `evaluator_feedback.md` — current iteration's feedback (overwritten each iter)
- `literature/constraints.md` — incremental constraint updates
- `literature/search_log.md` — cumulative search query log
- `phase_state.md` — current pipeline position (written by shell script)

## Final Outputs (written once in Phase 3, not updated)
- `planning/research_plan_final.md` — consolidated research program
- `planning/paper_structure.md` — section map for Paper Writer
- `planning/task_queue.md` — task list for Theory Builder / Empirical Agent
- `planning/novelty_claims.md` — contribution claims for verification
- `literature/threat_map_final.md` — clean, definitive threat map
- `literature/review.md` — prose review for LaTeX conversion
- `literature/notes.md` — structured paper summaries
- `model_equations.md` — all derivations and propositions
- `model_verifier_report.md` — Model Verifier assessment

## Iteration Archives (auto-generated by pipeline)
- `archives/evaluator_feedback_i{N}.md` — archived feedback from iteration N

## Phase 4 QA Loop State
- `self_reviews/` — review_theory.md, review_presentation.md, review_framing.md
- `improvement_task_queue.md`
