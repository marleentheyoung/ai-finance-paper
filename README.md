# Research Paper Toolkit

A multi-agent toolkit for drafting, reviewing, and revising academic research papers using [Claude Code](https://docs.anthropic.com/en/docs/claude-code). Write a one-page research idea or drop in an existing manuscript — the agents handle literature scanning, plan iteration, formal derivation, math verification, manuscript writing, and structured peer review.

Built for formal theory papers targeting top journal standards (JF / RFS / Econometrica), but adaptable to other fields.

---

## Three Workflows

This toolkit supports three distinct workflows. Each has its own detailed README:

| Workflow | What it does | Entry point |
|----------|-------------|-------------|
| **[Drafting](workflow/drafting/README.md)** | Takes a one-page research idea and produces a full LaTeX manuscript through a 4-phase pipeline (literature scan → plan iteration → execution → QA) | `context/research_context.md` → `./workflow/drafting/run_pipeline.sh` |
| **[Reviewing](workflow/reviewing/)** | Runs a structured peer review on an existing manuscript using an adaptive referee panel (editor desk-check → referee personas → parallel reports) | Drop a manuscript into `paper/` → run the review pipeline |
| **Review & Revise** | Combines both: reviews the manuscript, then revises it based on referee reports in an iterative loop | Run reviewing, then drafting QA phase on flagged sections |

All three workflows use the same devcontainer, agents, and shared `context/` directory. See [PIPELINE_MAP.md](PIPELINE_MAP.md) for the full architecture.

## Quick Start

### 1. Clone and open in the devcontainer

```bash
git clone <your-repo-url>
cd ai-finance-paper
code .
```

VS Code will detect the devcontainer configuration and show a popup: **"Reopen in Container"** — click it.

> **First build takes 3-5 minutes** (it installs Python, Node.js, LaTeX, and Claude Code inside the container). Subsequent opens are instant.

If you don't see the popup, open the Command Palette (`Cmd+Shift+P` / `Ctrl+Shift+P`) and run **"Dev Containers: Reopen in Container"**.

### 2. Install dependencies

Once inside the devcontainer terminal:

```bash
just install
```

This creates a Python virtual environment, installs all packages, and sets up pre-commit hooks. It also creates a `.env` file from the template if one doesn't exist yet.

### 3. Authenticate Claude Code

You need either a **Claude Pro/Max subscription** or an **Anthropic API key**. You don't need to configure this in advance — the pipeline scripts ask you on launch. To prepare:

- **Subscription:** Run `claude` in the devcontainer terminal and follow the browser login prompt to log in once.
- **API key:** Export `ANTHROPIC_API_KEY` in your host shell profile (e.g. `~/.zshrc`), then **open VS Code from that terminal** with `code .` so it inherits the variable. The devcontainer passes it through automatically.

### 4. Choose your workflow

| Goal | Next step |
|------|-----------|
| Draft a new paper from a research idea | Follow the **[Drafting Pipeline README](workflow/drafting/README.md)** |
| Review an existing manuscript | Follow the **[Reviewing Pipeline README](workflow/reviewing/)** |
| Review then revise iteratively | Start with reviewing, then use drafting QA to address issues |

## Directory Structure

```
context/                  # Research artifacts — agents read and write here
  research_context.md     # YOUR INPUT — the only file you write (drafting workflow)
  evaluation_criteria.md  # Scoring rubric (customise for your venue)
  self_reviews/           # QA loop review reports
  README.md               # File registry explaining each context file

paper/                    # LaTeX manuscript
  main.tex                # Document preamble (add \input lines for sections)
  references.bib          # Bibliography
  sections/               # Section .tex files (generated by Paper Writer)
  tables/                 # Table .tex files (generated by Empirical Agent)

code/                     # Python scripts for data and analysis
  verification/           # SymPy verification scripts (generated by Model Verifier)

workflow/                 # Pipeline orchestration
  drafting/               # Seed-to-paper drafting pipeline
    run_pipeline.sh       #   Automated pipeline runner (Phases 1-3)
    interactive_runbook.md#   Step-by-step prompts for interactive runs
    research-log.md       #   Append-only process log
    phase*_flowchart.md   #   Visual flowcharts per phase
  reviewing/              # Standalone manuscript review pipeline
    research-log.md       #   Review process log

skills/                   # Reusable prompt templates for agents
  academic-writing/       # LaTeX writing conventions
  manuscript-layout/      # Layout and formatting standards
  self-critique/          # Structured self-critique workflow
  economic-model-builder/ # Model derivation workflow
  literature-review-*/    # Literature search workflows (light/targeted/deep)
  empirical-design/       # Empirical pipeline (stub)

.claude/agents/           # Agent definitions (role, tools, modes)
archive/                  # Worked example from a prior project
```

## Agents

| Agent | Role | Workflows |
|-------|------|-----------|
| Research Director | Strategic planning, task decomposition, quality synthesis | Drafting |
| Literature Guardian | Novelty risk analysis, literature mapping (uses web search) | Drafting, Reviewing |
| Research Evaluator | Quality scoring against rubric, referee simulation | Drafting |
| Theory Builder | Formal model derivation, equilibrium solving | Drafting |
| Model Verifier | Mathematical verification via SymPy | Drafting |
| Paper Writer | LaTeX manuscript writing (4 sequential passes) | Drafting, Review & Revise |
| Theory/Presentation/Framing Lenses | Self-review critique (run in parallel) | Drafting (QA), Reviewing |
| Comprehensive Reviewer | Full referee simulation (5 modes) | Drafting (QA), Reviewing |
| Editor | Associate editor: desk-reject gate, referee panel construction | Reviewing |
| Referee Generator | Builds research-grounded referee personas | Reviewing |
| Referee | Runs 5-phase review process from a generated persona | Reviewing |
| Quality Reviewer | Editorial review: prose, presentation, journal conventions | Reviewing |

Each agent's full specification is in `.claude/agents/<name>.md`.

## Troubleshooting

### "Reopen in Container" doesn't appear

Make sure the [Dev Containers extension](https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-containers) is installed in VS Code, and that Docker Desktop is running.

### `just install` fails

- Make sure you're inside the devcontainer (the terminal prompt should show the container)
- If you see permission errors, run: `sudo chown -R vscode:vscode /home/vscode`

### Pipeline fails with "ANTHROPIC_API_KEY is not set"

This means you chose "API key" auth when the pipeline prompted you, but no key is in the environment. Either:
- Export `ANTHROPIC_API_KEY` in your host shell profile **before** opening the devcontainer, or
- Re-run the pipeline and choose "subscription" auth instead (this uses your browser login and doesn't need an API key)

### Pipeline fails silently or returns auth errors

If you have both a subscription and a stale API key set, the API key takes precedence and may fail. Re-run the pipeline and choose "subscription" — this unsets the API key so Claude Code uses your browser login instead.

### Pipeline stops at Model Verifier with "FAIL"

This means the formal model has issues the Theory Builder couldn't auto-fix. Check `context/model_verifier_report.md` for details. You may need to revise `context/research_context.md` (e.g., simplify scope constraints) and re-run from Phase 3.

### LaTeX compilation errors

- Make sure section files exist in `paper/sections/` before running `just render-paper`
- Check that `paper/main.tex` has the right `\input{sections/...}` lines
- Citation warnings are normal on first compile — run `just render-paper` twice

## Platform Notes

This pipeline is designed to run **inside the devcontainer**. Running it directly on macOS or Linux is possible but requires manual setup:

- Install Python 3.12+, Node.js 20+, and Claude Code CLI (`npm install -g @anthropic-ai/claude-code`)
- Install LaTeX (`texlive-full` or equivalent)
- Install `just` command runner
- Install `uv` package manager
- The pipeline script uses Linux-style `sed -i` — on macOS, you'd need `gsed` or adjust the commands

Using the devcontainer avoids all of this.
