# Research Paper Pipeline

A multi-agent pipeline for drafting academic research papers, targeting top journal standards (JF / RFS / Econometrica).

## What This Is

This repository is a **reusable template** for producing a formal theory paper using a coordinated team of AI agents. Each agent has a bounded role, and all inter-agent communication happens through shared context files.

## Architecture

The pipeline has four phases:

1. **Pre-loop** — Human provides `context/research_context.md`; Literature Guardian scans for threats; Research Director produces initial plan
2. **Planning loop** — Iteratively refine the plan until it passes a quality gate (score >= 4.0 on a 1-5 scale, max 5 iterations)
3. **Post-loop execution** — Derive models, verify math, evaluate outputs, write the manuscript in four sequential passes
4. **Quality assurance loop** — Three self-review lenses identify issues; Research Director synthesises improvements; agents execute fixes

See `PIPELINE_MAP.md` for the full architecture, flowcharts, and agent catalogue.

## Getting Started

1. **Clone this repo** and open in the devcontainer
2. **Install dependencies:** `just install`
3. **Fill in your research idea:** edit `context/research_context.md` with your research question, mechanism, contributions, and scope constraints
4. **Run the pipeline:** `./workflow/run_pipeline.sh`

The pipeline runs autonomously, producing all intermediate artifacts in `context/` and the final manuscript in `paper/sections/*.tex`.

## Directory Structure

```
context/                  # Research artifacts (agents read/write here)
  research_context.md     # YOUR INPUT — the "idea seed"
  evaluation_criteria.md  # Scoring rubric (customise for your venue)
  self_reviews/           # QA loop review reports
  README.md               # File registry

paper/                    # LaTeX manuscript
  main.tex                # Document preamble and section includes
  references.bib          # Bibliography
  sections/               # Section .tex files (generated by Paper Writer)
  tables/                 # Table .tex files (generated by Empirical Agent)

code/                     # Python scripts
  verification/           # SymPy verification scripts (generated by Model Verifier)

workflow/                 # Pipeline orchestration
  run_pipeline.sh         # Overnight pipeline runner
  research-log.md         # Append-only process log

skills/                   # Skill definitions (reusable prompts)
  academic-writing/       # LaTeX writing conventions
  manuscript-layout/      # Layout and formatting standards
  self-critique/          # Structured self-critique workflow
  economic-model-builder/ # Model derivation workflow
  literature-review-*/    # Literature search workflows
  empirical-design/       # Empirical pipeline (stub)

.claude/agents/           # Agent definitions
  research-director.md    # Principal investigator
  literature-guardian.md  # Novelty risk analyst
  research-evaluator.md   # Skeptical coauthor
  theory-builder.md       # Formal model developer
  model-verifier.md       # Mathematical auditor
  paper-writer.md         # Manuscript writer
  theory-lens.md          # QA: theory self-review
  presentation-lens.md    # QA: presentation self-review
  framing-lens.md         # QA: framing self-review
  comprehensive-reviewer.md # QA: full referee simulation
```

## Agents

| Agent | Role | Phase |
|-------|------|-------|
| Research Director | Strategic planning, task decomposition, improvement synthesis | 1, 2, 3, 4 |
| Literature Guardian | Novelty risk analysis, literature mapping | 1, 2, 3, 4 |
| Research Evaluator | Quality scoring, referee simulation | 2, 3 |
| Theory Builder | Formal model derivation, equation improvement | 3, 4 |
| Model Verifier | Mathematical verification via SymPy | 3 |
| Paper Writer | LaTeX manuscript writing (4 sequential passes) | 3, 4 |
| Theory/Presentation/Framing Lenses | Self-review critique | 4 |
| Comprehensive Reviewer | Full referee simulation (5 modes) | 3, 4 |
